# Repository Map

| Area | Path | Purpose |
|---|---|---|
| Application source | `src/` | Core Node modules for transcript ingestion, parsing, pricing, and the in-memory read model. `parseClaude.js` and `parseCodex.js` normalize provider JSONL files into Sessions, Human requests, LLM calls, and Token buckets; `pricing.js` resolves model rates/context windows and computes Estimated cost; `store.js` enriches parsed Sessions and produces API projections; `watcher.js` scans and watches transcript directories. |
| Frontend source | `public/` | Static local dashboard served by Express. `index.html` defines the shell, `styles.css` owns visual styling and themes, and `app.js` polls the local API, filters Sessions, renders summary views, and displays Human request and LLM call details. |
| Tests | `test/llm-insights.test.js` | Node test coverage for Claude Code and Codex parser normalization plus Store projection/enrichment behavior for LLM-call insight fields and cost-driver derivation. |
| Entry points | `start.js`, `server.js` | `start.js` is the local bootstrap command used by `npm start`; it installs missing runtime dependencies before loading the app. `server.js` composes Pricing, Store, Watcher, static assets, REST routes, source-directory configuration, listen host/port, and best-effort browser opening. |
| Configuration | `package.json`, `package-lock.json`, `.gitignore`, `docs/development-commands.md` | npm scripts, dependency declarations, Node engine requirement, locked dependency graph, ignored runtime/tooling paths, and supported local development commands. |
| Product and design docs | `design/`, `design/handoff/` | Product brief, information architecture, user flows, UI spec, accessibility notes, component/screen inventory, visual direction, states/edge cases, implementation brief, and handoff tokens/themes for the dashboard experience. |
| Project documentation | `README.md`, `CONTEXT.md`, `AGENTS.md`, `CLAUDE.md`, `docs/` | Human and agent orientation. `README.md` is the concise public project entry point; `CONTEXT.md` and agent instruction files route work; `docs/` contains current conventions, canonical domain language, development commands, this map, and normative technical documentation. |
| Normative technical docs | `docs/normative/` | Accepted architecture, decisions, and external integration contracts. Read `docs/normative/README.md` first, then only the relevant routed documents. |
| Documentation assets | `docs/assets/` | Static image assets (app screenshots such as `Stats.png`, `Session_Overview.png`, `Human_Request.png`) embedded by relative path in `README.md` and other docs. Replace/rename in place when the UI changes; keep references in sync. |
| Generated or vendored content | `node_modules/`, `.cache/litellm_prices.json`, `.codegraph/` | `node_modules/` is installed by npm and ignored; do not hand-edit. `.cache/litellm_prices.json` is a runtime LiteLLM price-cache written by `src/pricing.js`; delete/regenerate rather than hand-maintain. `.codegraph/` is local code-index state and ignored. |
| Migrations or infrastructure | None | The app has no database migrations, hosted infrastructure, deployment manifests, or schema-management layer; runtime Session data is in-memory and transcript/price data comes from local files or the LiteLLM fetch/cache path. |
