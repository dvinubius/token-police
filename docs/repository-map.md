# Repository Map

| Area | Path | Purpose |
|---|---|---|
| Application source | `src/` | Core Node modules for transcript ingestion, parsing, pricing, and the in-memory read model. `parseClaude.js` and `parseCodex.js` normalize provider JSONL files into Sessions, Human requests, LLM calls, and Token buckets; `parseShared.js` owns genuinely shared parsing utilities; `pricing.js` resolves model rates/context windows and computes Estimated cost; `store.js` enriches parsed Sessions and produces API projections; `watcher.js` scans and watches transcript directories. |
| Frontend source | `frontend/` | Svelte + Vite dashboard source. `frontend/index.html` defines the HTML shell; `frontend/src/` contains the component tree, reactive store, extracted pure view helpers, API client, and global stylesheet. |
| Demo data build | `demo/` | Generator for the hosted static demo. `dataset.js` is the authored spec (fictitious projects, Human request prompts, LLM call counts, cost-spike shapes); `transcripts.js` renders it into provider-shaped `.jsonl` fixtures laid out like the real transcript directories; `build.js` parses those fixtures with the real parsers, `Pricing`, and `Store`, and freezes the API projections into `dist/demo/`. Nothing here is loaded by the local server. |
| Tests | `test/` | Node tests for provider parser normalization, zero-call Human requests, subagent hierarchy, Store enrichment/projections/summary behavior, and framework-independent frontend formatting, grouping, filtering, sorting, visibility, and totals helpers. `demo-dataset.test.js` additionally runs the demo generator through the real parsers and asserts the hosted demo still demonstrates both providers, multi-model Sessions, subagent hierarchies, zero-call Human requests, and every cost driver. |
| Entry points | `start.js`, `server.js` | `npm start` builds the frontend and then invokes `start.js`; the bootstrap installs missing runtime dependencies before loading `server.js`. `server.js` composes Pricing, Store, Watcher, static assets, REST routes, source-directory configuration, listen host/port, and best-effort browser opening. |
| Configuration | `package.json`, `package-lock.json`, `vite.config.mjs`, `.gitignore`, `docs/development-commands.md` | npm scripts, dependency declarations, Node engine requirement, locked dependency graph, Vite build/proxy configuration, ignored runtime/tooling paths, and supported local development commands. |
| Product and design docs | `design/`, `design/handoff/` | Product brief, information architecture, user flows, UI spec, accessibility notes, component/screen inventory, visual direction, states/edge cases, and an implementation-aligned visual-system handoff. Historical pre-liquid token/theme artifacts are retained as design-history snapshots; `frontend/src/styles.css` is authoritative for implemented values. |
| Project documentation | `README.md`, `CONTEXT.md`, `AGENTS.md`, `CLAUDE.md`, `docs/` | Human and agent orientation. `README.md` is the concise public project entry point; `CONTEXT.md` and agent instruction files route work; `docs/` contains current conventions, canonical domain language, development commands, this map, and normative technical documentation. |
| Normative technical docs | `docs/normative/` | Accepted architecture, decisions, and external integration contracts. Read `docs/normative/README.md` first, then only the relevant routed documents. |
| Documentation assets | `docs/assets/` | Light/dark app screenshots retained as optional documentation evidence (`Stats_*`, `Session_Overview_*`, `Human_Request_*`). They are not runtime assets; replace or rename them in place when reused after UI changes. |
| Generated or vendored content | `dist/`, `node_modules/`, `.cache/litellm_prices.json`, `.codegraph/`, `.demo-build/` | `dist/` is generated by `npm run build` and served by Express; do not hand-edit. `dist/demo/` is written only by `npm run build:demo`, and `.demo-build/` holds that build's intermediate `.jsonl` fixtures; both are regenerated, never edited. `node_modules/` is installed by npm and ignored. `.cache/litellm_prices.json` is a runtime LiteLLM price-cache written by `src/pricing.js`; delete/regenerate rather than hand-maintain. `.codegraph/` is local code-index state and ignored. |
| Migrations or infrastructure | None | The app has no database migrations, hosted infrastructure, deployment manifests, or schema-management layer; runtime Session data is in-memory and transcript/price data comes from local files or the LiteLLM fetch/cache path. |
